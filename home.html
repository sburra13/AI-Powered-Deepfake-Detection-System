FAQ

Common questions

Everything you need to know about how DeepFake works.

What types of deepfakes can DeepFake detect?
▾
Our DeepFake model is trained on FaceForensics++ data and can detect face-swap deepfakes, GAN-generated faces (e.g. StyleGAN, DALL-E), and face re-enactment attacks. It performs best on frontal face images with good lighting.
How accurate is the model?
▾
Our Xception-based model achieves 97.4% accuracy and a 99.1% AUC score on our validation set. Results may vary on heavily compressed or low-resolution media.
Is my uploaded image or video stored?
▾
No. Uploaded files are deleted immediately after inference. We do not store, log, or share any media you upload. Everything runs locally on our server and is discarded in the same request.
What file formats are supported?
▾
Images: JPEG, PNG, WebP, BMP. Videos: MP4, MOV, AVI, MKV. Maximum file size is 50MB for images and 200MB for videos. We recommend at least 224×224 pixel resolution for best results.
Why does the model need to detect a face?
▾
Deepfake manipulation almost always targets the face. By isolating the face region first, the model focuses on the exact pixels where manipulation occurs, dramatically improving accuracy over whole-image analysis.
Can I use the API in my own project?
▾
The backend is a FastAPI service. You can call the /predict endpoint directly with a multipart POST request. See the GitHub repository for full API documentation and usage examples.
About Us

Built for the fight
against misinformation

DeepFake is our Masters project exploring the intersection of computer vision and media authenticity.

Our Mission

As deepfake technology becomes increasingly accessible, the ability to distinguish real from synthetic media is critical. Our DeepFake provides a fast, accessible tool to help journalists, researchers, and the general public verify the authenticity of visual content.

Our Technology

We fine-tuned Google's Xception architecture — the same model validated on FaceForensics++ — using strong data augmentation including JPEG compression simulation, which mirrors real-world deepfake distribution. The model is served via a FastAPI backend with sub-second inference.
